# Goblin Mode — Decisions Log & Open Questions

*The single source of truth a returning solo dev checks first. Everything resolved lives in the **Decided** table; everything still open lives in **Open questions**. This is a living document, not a contract — revisit it at every milestone and move items from Open to Decided (with the answer and date) as the prototype teaches you what's true. When the other docs and this log disagree, fix the doc to match the log.*

A *decision* here means a creative or technical choice that more than one doc depends on. Most were accepted best-practice recommendations; three were the developer's own calls (multiplayer, perspective, price) — flagged **[dev call]** below.

## How to use this log

- **Revisit at every milestone review** (and at the M2 gate especially). This is the first file a returning dev should open.
- **When an Open item is resolved,** move it to the **Decided** table with a concise resolution and the date you decided it; delete it from **Open**. Update the affected docs to match (the docs defer to this log, not the other way round).
- **When a new cross-cutting decision is made,** add a row to **Decided** rather than burying it in a single doc.
- **If the docs and this log ever disagree,** the log is canonical for *what was decided*; fix the doc. (The docs remain canonical for the detailed *design itself*.)
- Treat the whole thing as a **living hypothesis** — the same spirit as the GDD. Nothing here is too settled to revise if the prototype proves it wrong; just record the revision and the date.
